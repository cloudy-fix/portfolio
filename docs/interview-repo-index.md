## Public Cleanup Standard

- Use lowercase kebab-case for new repository names.
- Put the strongest cloud and DevOps work first in README/profile links.
- Keep descriptions concise and recruiter-readable.
- Add topics to every public repository so GitHub search and profile scanning look organized.
- Keep personal documents outside repositories.
- Keep private repos private when they include unpublished client, business, or sensitive project details.
